# 第 17 章 RT-2 与机器人学习：从语言模型到机器人控制

## 17.1 RT-2 背景与贡献

### 17.1.1 研究动机

RT-2（Robotic Transformer 2）是 Google DeepMind 在 2023 年提出的开创性工作，其核心动机是：

1. **利用大规模语言模型的知识**
   - LLM 在预训练中学习了丰富的世界知识
   - 如何将这种知识迁移到机器人控制？

2. **统一视觉 - 语言 - 动作建模**
   - 打破感知与控制的界限
   - 实现端到端的机器人策略学习

3. **零样本泛化能力**
   - 面对新任务、新物体时能够泛化
   - 减少对特定任务数据的依赖

### 17.1.2 主要贡献

RT-2 的核心贡献包括：

1. **视觉 - 语言 - 动作（VLA）模型**
   - 将机器人控制表述为序列预测问题
   - 统一处理视觉、语言和动作 token

2. **大规模数据训练**
   - 结合机器人数据和网络数据
   - 学习通用的机器人策略

3. **涌现的零样本能力**
   - 语义理解：理解抽象指令
   - 推理能力：执行多步推理任务
   - 数学计算：进行简单计算后执行动作

## 17.2 从语言模型到机器人控制

### 17.2.1 核心思想

RT-2 的关键洞察是：**机器人控制可以被视为另一种形式的语言建模**。

```
传统 LLM:     文本输入 → 文本输出
RT-2 VLA:     图像 + 文本 → 动作（编码为 token）
```

### 17.2.2 Token 化动作

RT-2 将连续动作离散化为 token：

```python
# 动作 token 化示例
# 将连续值映射到离散 bin

# 原始动作：关节角度 [-π, π]
joint_angle = 0.5  # 弧度

# 离散化：分为 256 个 bin
num_bins = 256
action_bin = int((joint_angle + np.pi) / (2 * np.pi) * num_bins)
# action_bin ∈ [0, 255]

# 每个动作维度用一个 token 表示
action_tokens = [bin_1, bin_2, ..., bin_7]  # 7 自由度
```

### 17.2.3 统一序列格式

RT-2 使用统一的序列格式：

```
[图像 patch tokens] + [文本 tokens] + [动作 tokens]
```

示例：
```
[img_1][img_2]...[img_N] [拾] [取] [杯] [子] [act_1][act_2]...[act_7]
```

### 17.2.4 与 LLM 的类比

| 组件 | LLM | RT-2 |
|------|-----|------|
| 输入 | 文本序列 | 图像 + 文本 |
| 处理 | Self-Attention | Self-Attention |
| 输出 | 下一个词 | 下一个动作 token |
| 训练 | 语言建模 | 行为克隆 |

## 17.3 机器人 Transformer 架构

### 17.3.1 架构概述

RT-2 基于预训练的视觉 - 语言模型（如 PaLI、PaLM-E），添加动作预测头：

```
┌────────────────────────────────────────────────────────────┐
│                      RT-2 架构                              │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  ┌─────────────┐    ┌─────────────┐                        │
│  │   图像输入   │    │   文本输入   │                        │
│  │  (摄像头)   │    │   (指令)    │                        │
│  └──────┬──────┘    └──────┬──────┘                        │
│         │                  │                                │
│         ▼                  ▼                                │
│  ┌─────────────┐    ┌─────────────┐                        │
│  │  ViT 编码器  │    │ 文本编码器  │                        │
│  └──────┬──────┘    └──────┬──────┘                        │
│         │                  │                                │
│         └────────┬─────────┘                                │
│                  ▼                                          │
│         ┌─────────────────┐                                 │
│         │  Transformer    │  预训练 VLM 骨干                 │
│         │  (冻结/微调)    │                                 │
│         └────────┬────────┘                                 │
│                  │                                          │
│                  ▼                                          │
│         ┌─────────────────┐                                 │
│         │  动作预测头     │  新增                           │
│         │  (Action Head)  │                                 │
│         └────────┬────────┘                                 │
│                  │                                          │
│                  ▼                                          │
│         ┌─────────────────┐                                 │
│         │   动作 Tokens   │                                 │
│         └─────────────────┘                                 │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 17.3.2 关键设计决策

1. **使用预训练 VLM**
   - 利用大规模预训练的知识
   - 减少机器人数据需求

2. **冻结大部分参数**
   - 只微调少量参数
   - 保持预训练知识

3. **动作作为输出 token**
   - 与文本 token 共享词表
   - 或独立的动作词表

### 17.3.3 注意力机制

RT-2 使用标准的 Transformer 自注意力：

```python
# 自注意力计算
Attention(Q, K, V) = softmax(QK^T / √d) V

# 在 RT-2 中：
# Q, K, V 来自图像 token 和文本 token 的混合序列
# 动作 token 只作为输出，不参与注意力
```

## 17.4 大规模机器人数据训练

### 17.4.1 数据来源

RT-2 训练数据来自多个来源：

1. **机器人数据（~50%）**
   - Google 内部机器人采集
   - 公开数据集（Bridge、RT-1 等）
   - 约 100,000+ 条轨迹

2. **网络数据（~50%）**
   - 图像 - 文本对
   - 保持 VLM 的视觉 - 语言能力
   - 防止灾难性遗忘

### 17.4.2 数据格式

```python
# RT-2 训练数据示例
data_sample = {
    "images": [...],           # 图像序列
    "instruction": "拿起红色的杯子",  # 文本指令
    "actions": [...],          # 动作序列（已 token 化）
    "success": True            # 是否成功
}
```

### 17.4.3 训练目标

RT-2 使用行为克隆（Behavior Cloning）训练：

```python
# 损失函数：预测动作 token 的交叉熵
loss = CrossEntropyLoss(predicted_action_tokens, expert_action_tokens)

# 只监督动作 token，不监督图像和文本 token
# 因为图像和文本是输入，动作是输出
```

### 17.4.4 训练策略

1. **两阶段训练**
   - 阶段 1：在 VLM 上预训练
   - 阶段 2：在机器人数据上微调

2. **渐进式微调**
   - 先解冻少量层
   - 逐步解冻更多层

3. **数据混合**
   - 保持一定比例的网络数据
   - 防止遗忘预训练知识

## 17.5 零样本迁移能力

### 17.5.1 什么是零样本迁移？

零样本迁移（Zero-Shot Transfer）指模型能够：
- 执行训练时未见过的任务
- 操作训练时未见的物体
- 理解新的语言指令

### 17.5.2 RT-2 的零样本能力

RT-2 展现了多种零样本能力：

1. **语义理解**
   - 指令："把可乐递给我"
   - 模型识别"可乐"并抓取正确物体

2. **推理能力**
   - 指令："如果桌子是乱的，就整理它"
   - 模型判断桌子状态并执行相应动作

3. **数学计算**
   - 指令："移动到 x=5+3 的位置"
   - 模型计算 5+3=8 并移动到 x=8

4. **多语言支持**
   - 指令可以用多种语言
   - 利用 VLM 的多语言能力

### 17.5.3 能力来源

RT-2 的零样本能力来自：

1. **预训练 VLM 的知识**
   - 世界知识
   - 语义理解
   - 推理能力

2. **大规模数据训练**
   - 多样化的任务
   - 多样化的物体

3. **统一的建模范式**
   - 视觉、语言、动作的统一表示
   - 便于知识迁移

## 17.6 实验结果与分析

### 17.6.1 评估指标

RT-2 使用以下指标评估：

1. **任务成功率（Success Rate）**
   - 完成任务的比例

2. **零样本泛化（Zero-Shot Generalization）**
   - 新物体、新指令的成功率

3. **与基线对比**
   - RT-1
   - 其他 VLA 模型

### 17.6.2 主要结果

RT-2 在多个基准上取得 SOTA：

| 任务类型 | RT-1 | RT-2 | 提升 |
|---------|------|------|------|
| seen objects | 65% | 78% | +13% |
| unseen objects | 32% | 50% | +18% |
| reasoning tasks | 15% | 45% | +30% |
| math tasks | 5% | 35% | +30% |

### 17.6.3 消融实验

RT-2 论文进行了多项消融实验：

1. **预训练 VLM 的重要性**
   - 无预训练：性能显著下降
   - 证明预训练知识的价值

2. **数据混合比例**
   - 纯机器人数据：过拟合
   - 50-50 混合：最佳平衡

3. **动作 token 化方式**
   - 不同离散化粒度
   - 256 bins 效果最佳

### 17.6.4 失败案例分析

RT-2 的局限性：

1. **长序列任务**
   - 多步任务容易出错
   - 误差累积问题

2. **精细操作**
   - 需要高精度的任务
   - 离散化限制

3. **动态环境**
   - 快速变化的场景
   - 实时性要求高

## 17.7 OpenVLA 开源方案

### 17.7.1 OpenVLA 简介

OpenVLA 是斯坦福大学开源的 VLA 模型，基于 RT-2 思想：

- **开源代码和模型**
- **基于 LLaMA 架构**
- **支持多种机器人平台**

### 17.7.2 OpenVLA 特点

1. **开源透明**
   - 完整代码公开
   - 预训练模型可下载

2. **易于使用**
   - 简单的 API
   - 详细的文档

3. **社区驱动**
   - 持续更新
   - 社区贡献

### 17.7.3 与 RT-2 的对比

| 特性 | RT-2 | OpenVLA |
|------|------|---------|
| 开源 | 否 | 是 |
| 基础模型 | PaLI/PaLM-E | LLaMA |
| 数据 | 私有 | 公开数据集 |
| 可复现性 | 低 | 高 |
